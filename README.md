# Trading Platform

Multi-tenant crypto algorithmic trading platform. Binance WebSocket ticks flow through a Kafka event pipeline (Python) into OMS/EMS microservices (C# .NET 8) backed by PostgreSQL + TimescaleDB, with full observability via Prometheus, Grafana, and Jaeger.

## Architecture

```mermaid
flowchart TD
    BW["Binance WebSocket\n(100 ms ticks)"]
    EC["Exchange Connector\n(Python)"]
    CA["Candle Aggregator\n(Python ¬∑ 5 m OHLCV)"]
    SP["Signal Pipeline\n(Python)"]
    ML["MLSignalSource\nLightGBM"]
    LLM["LLMSignalSource\nClaude / GPT"]
    SR["Signal Router\n(Python ¬∑ per-tenant)"]
    OMS["OMS\n(C# .NET 8)"]
    EMS["EMS\n(C# .NET 8)"]
    BRest["Binance REST\n(testnet)"]

    BW -->|market-data-raw| EC
    EC -->|market-data-candles| CA
    CA -->|candles| SP
    SP --> ML
    SP --> LLM
    ML -->|"trade-signals (source=ml)"| SR
    LLM -->|"trade-signals (source=llm)"| SR
    SR -->|order-requests| OMS
    OMS -->|validated-orders| EMS
    EMS --> BRest
    EMS -->|order-fills| OMS
```

Signal sources are pluggable ‚Äî select via `SIGNAL_SOURCE=ml|llm|both`. LLM signals are generated by Claude or GPT analysing recent candle data and technical indicators, running at a configurable slower cadence (default 15 min) to manage API cost.

## Quick Start

```bash
# 1. Configure environment
cp .env.example .env
# Edit .env ‚Äî set POSTGRES_PASSWORD, BINANCE_API_KEY, BINANCE_SECRET_KEY

# 2. Start the full stack
cd infra
docker-compose up -d

# 3. Seed tenant data
docker exec -i infra-postgres-1 psql -U trading -d trading \
  < ../scripts/seed-tenants.sql

# 4. Verify Kafka flow
bash ../scripts/verify-kafka-flow.sh
```

Or use the dev script:
```bash
bash scripts/start-dev.sh
```

## Environment Variables

| Variable                  | Required | Default                              | Description                         |
|---------------------------|----------|--------------------------------------|-------------------------------------|
| `POSTGRES_PASSWORD`       | Yes      | ‚Äî                                    | PostgreSQL password                 |
| `BINANCE_API_KEY`         | Yes      | ‚Äî                                    | Binance (testnet) API key           |
| `BINANCE_SECRET_KEY`      | Yes      | ‚Äî                                    | Binance (testnet) secret key        |
| `BINANCE_BASE_URL`        | No       | `https://testnet.binance.vision`     | Use testnet for local dev           |
| `KAFKA_BROKERS`           | No       | `kafka:9092`                         |                                     |
| `TRADE_SYMBOLS`           | No       | `btcusdt,ethusdt`                    | Comma-separated symbols             |
| `CANDLE_TIMEFRAME_MINUTES`| No       | `5`                                  |                                     |
| `SIGNAL_MIN_CONFIDENCE`        | No       | `0.7`                            | Min ML confidence to emit a signal       |
| `SIGNAL_TTL_SECONDS`           | No       | `90`                             | Signal staleness cutoff                  |
| `SIGNAL_INTERVAL_SECONDS`      | No       | `300`                            | ML signal generation cadence             |
| `SIGNAL_SOURCE`                | No       | `ml`                             | `ml`, `llm`, or `both`                   |
| `LLM_PROVIDER`                 | No       | `anthropic`                      | `anthropic` or `openai`                  |
| `ANTHROPIC_API_KEY`            | Only if `LLM_PROVIDER=anthropic` | ‚Äî | Anthropic API key               |
| `OPENAI_API_KEY`               | Only if `LLM_PROVIDER=openai`    | ‚Äî | OpenAI API key                  |
| `LLM_SIGNAL_INTERVAL_SECONDS`  | No       | `900`                            | LLM signal cadence (s) ‚Äî manage API cost |
| `LLM_CANDLE_WINDOW`            | No       | `20`                             | Candles sent in LLM prompt               |
| `LOG_LEVEL`                    | No       | `INFO`                           |                                          |
